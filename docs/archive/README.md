## How to use this archive

- Treat these files as historical context, not as active policy.
- If an archived note conflicts with the current README, LICENSE, SECURITY, CONTRIBUTING, ROADMAP, or CHANGELOG, the current docs win.
- Keep public-facing examples free of private hosts, private emails, personal names, and local paths.

## Suggested contents

- Historical commercial planning notes.
- Drafts that describe prior release lanes or support flows.
- Imported summaries that are kept only for traceability.

## Public release rule

Do not cite archived material as evidence of current product behavior, release policy, pricing, support, or production readiness.
